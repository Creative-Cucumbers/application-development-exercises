# 授業設計・スライド制作用プロンプトメモ

今後の授業準備で、AI に相談するときのたたき台。
必要に応じて科目名、対象学年、授業時間、到達目標を差し替えて使う。

## 1. 授業全体の方針を考える

### 授業の方向性を整理したいとき

```text
この授業のテーマは「[テーマ]」です。
対象は [対象者] で、1コマ [時間] 分、全 [回数] 回です。
授業の狙いは [狙い] です。

この授業について、以下を整理してください。
- 学生に最終的に身につけてほしいこと
- 各回で扱うとよい内容
- 難しすぎる点、つまずきやすい点
- 実習中心にする場合の流れ

教育現場で実施しやすい、現実的な提案にしてください。
```

### 各回のバランスを考えたいとき

```text
以下の授業回の内容案があります。
[内容案を貼る]

これをもとに、
- 導入
- 説明
- 実習
- 振り返り
- 提出課題

のバランスを見直してください。
時間配分の目安も提案してください。
説明が多すぎる箇所、実習が重すぎる箇所があれば指摘してください。
```

## 2. 実習課題を考える

### 提出課題を具体化したいとき

```text
この授業回の内容は以下です。
[授業内容を貼る]

この内容に対応する提出課題を考えてください。
条件:
- 成績評価のため記録が残ること
- 画像、写真、短い文章のいずれかを含めること
- 学生が過度に負担にならないこと
- AIを使ってもよい前提にすること

提出物の例文も入れてください。
```

### 説明文の例を作りたいとき

```text
以下の実習について、学生向けの短い説明文の例を 3 つ作ってください。
[実習内容]

条件:
- 1つ 2〜4文
- 入力、処理、出力がわかること
- 学生がそのまま参考にできる自然な文にすること
```

## 3. AI活用の進め方を考える

### いきなり実装ではなく深掘りさせたいとき

```text
この授業では、学生に AI へいきなりコード生成を頼むのではなく、
アイデアを深掘りしてから実装に進む流れを体験させたいです。

以下のテーマについて、
- 深掘り段階で使えるプロンプト
- アイデア整理のためのプロンプト
- 最後に実装を頼むプロンプト

を順番に作ってください。

テーマ:
[テーマ]
```

### 教員向けにAI利用方針を整理したいとき

```text
以下の授業内容で、学生に AI を使わせるときの方針を整理したいです。
[授業内容]

以下を簡潔にまとめてください。
- AI を使う目的
- AI に任せすぎないための注意点
- 学生に確認させたいポイント
- 提出物に残させるとよい記録
```

## 4. スライド構成を作る

### readme や授業メモからスライド案を作りたいとき

```text
以下の授業メモをもとに、講義スライドの構成案を作ってください。
[授業メモを貼る]

条件:
- 1スライド1メッセージを基本にする
- 導入、概念説明、実習、提出課題、まとめを含める
- スライドごとにタイトル案をつける
- 長文ではなく、箇条書き中心にする
```

### スライドが重いか確認したいとき

```text
以下は授業スライド原稿です。
[スライド原稿を貼る]

以下の観点でレビューしてください。
- 1枚あたりの情報量が多すぎないか
- 画像と文章のバランスはよいか
- 学生にとってわかりにくい表現はないか
- 分割した方がよいスライドはどこか

必要なら、分割案も出してください。
```

## 5. 素材集めを考える

### 必要な画像や図を洗い出したいとき

```text
以下のスライド構成に対して、必要な素材を洗い出してください。
[スライド構成を貼る]

素材ごとに
- 何のために必要か
- 画面キャプチャでよいか
- 写真が必要か
- 図解が必要か
- どのスライドで使うか

を整理してください。
```

### 参考事例を選びたいとき

```text
この授業テーマに合う参考事例を 5 個挙げてください。
テーマ: [テーマ]

条件:
- 学生に直感的に伝わりやすい
- スライドで紹介しやすい
- 入力、処理、出力の観点で説明しやすい

各事例について、
- どこが授業に向いているか
- どのスライドで使いやすいか

も書いてください。
```

## 6. HTML / アプリ制作デモを考える

### デモ用プロトタイプをAIに作らせたいとき

```text
授業デモ用に、1ファイルで動く簡単なプロトタイプを作りたいです。

要件:
- ファイルは index.html 1つ
- ブラウザでそのまま動く
- 見た目はシンプルでよい
- 学生が読んで理解しやすい
- コメントを適度に入れる

作りたいもの:
[作りたいもの]

必要なら、あとで機能追加しやすい構成にしてください。
```

### センサー入力とブラウザ表現をつなぐデモを考えたいとき

```text
micro:bit などのセンサー入力をブラウザ表現につなぐ授業デモを考えています。
テーマは [テーマ] です。

以下を提案してください。
- デモのコンセプト
- 必要な入力
- ブラウザ側の表現
- 最小構成の実装方針
- 学生に説明しやすいポイント
```

## 7. レビューしてもらう

### 先生目線でのチェックを頼みたいとき

```text
以下の授業案を、大学授業として現実的かどうかレビューしてください。
[授業案を貼る]

特に以下を見てください。
- 情報量が多すぎないか
- 1コマの中で無理がないか
- 実習と提出課題の対応が取れているか
- 初学者に厳しすぎる箇所はないか

改善点を優先度つきで出してください。
```

### スライド完成前の最終確認

```text
以下のスライド原稿を最終確認してください。
[原稿を貼る]

確認したい点:
- わかりやすさ
- 情報量
- 誤解を生みそうな表現
- 実習指示の明確さ
- 提出課題の具体性

必要な修正だけを簡潔に指摘してください。
```
