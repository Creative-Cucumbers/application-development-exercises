# Day2 フィジカルコンピューティング：基礎・入力・出力

[TOC]

## この日の位置づけ

Day1でAI時代のものづくりの全体像を掴んだ次のステップとして、実際に手を動かしてフィジカルコンピューティングの基礎を体験する。micro:bit と MakeCode を使い、「デジタルが物理世界とつながる」感覚をつかむ。

1コマ目で概念・環境構築・入力センサーを扱い、2コマ目で出力（LED・スピーカー）まで一通り体験する。1日で入力から出力まで触れることで、次回（Day3）の応用制作につながるイメージを持ちやすくする。

授業はハイブリッド形式で実施する。現地参加者は実機の micro:bit を使い、リモート参加者は MakeCode のシミュレーターで同じ実習を行う。

## 到達目標

- フィジカルコンピューティングの概念と代表的な活用事例を説明できる
- micro:bit と MakeCode の基本操作を習得し、自分でプログラムを書いて実行できる
- AI（Copilot・ChatGPT等）を使ってMakeCodeのJavaScriptコードを生成・修正できる
- AIが生成したコードが何をしているか、自分の言葉で説明できる
- micro:bit のボタン・加速度センサー・温度センサーのデータを取得できる
- LEDマトリクスとスピーカーを使った出力プログラムを組める
- AIがフィジカルコンピューティングの表現をどう拡張しているか、自分の言葉で話せる

## アジェンダ案

### 0. アイスブレイク

- テーマ：「身の回りで『センサーが動かしているもの』を1つ挙げてみよう」
  - 例：自動ドア、スマホの画面回転、ゲームコントローラーの振動など
- 現地・リモートともにチャットや口頭で共有（2〜3分）
- 教員がいくつかピックアップして簡単にコメント
- 狙い：フィジカルコンピューティングへの興味付け、リモート参加者も最初から発言できる空気を作る

### 1. フィジカルコンピューティングとは

- デジタルと物理世界をつなぐとはどういうことか
- 代表的な事例紹介
  - インタラクティブアート（teamLab, Zimoun など）
  - スマート家電・IoTデバイス
  - 体験型インスタレーション
  - ロボット制御（センサーで障害物を検知して動きを変える、腕の動きをモーターに伝えるなど）
  - 自動車・モビリティ（カメラ・LiDARで周囲を認識して走行・停止を判断する自動運転、衝突検知センサーなど）
  - AIとセンサーを組み合わせた作品（音声認識 + LED、顔認識 + 物理フィードバックなど）
- AI時代のフィジカルコンピューティング
  - センサーデータをAIモデルに渡すことで、より豊かなインタラクションが可能に
  - コード生成AIを使えば、プログラミング経験が少なくてもプロトタイプを作れる
  - 表現のアイデアを先に決め、実装はAIと協力する開発スタイル

### 2. micro:bit と MakeCode の基本・チュートリアル

- micro:bit の各部位の確認
  - 5×5 LEDマトリクス
  - ボタン A / B
  - 加速度センサー、温度センサー、光センサー、コンパス
  - スピーカー（v2）
  - エッジコネクタ（外部接続ピン）
- MakeCode の基本操作
  - ブラウザで開く（makecode.microbit.org）
  - ブロックとJavaScriptの切り替え
  - シミュレーターで動作確認（リモート参加者はここで実習を完結できる）
  - micro:bit への書き込み手順（USBドライブとして転送）※現地参加者のみ
- MakeCode の公式チュートリアルをいくつか全員で一緒に試す
  - チュートリアルを進めながら「このブロックは何をしているか」を確認していく

### 3. AIを使ったコード生成の実践

- チュートリアルで動かしたプログラムをJavaScriptで見てみる
  - ブロックとJavaScriptが対応していることを確認する
- 次は自分でAIにコードを書かせてみる
  - プロンプト例：「MakeCode JavaScript で、ボタンAを押すたびにカウントアップしてLEDに表示するコードを書いて」
  - プロンプト例：「MakeCode JavaScript で、micro:bit を振るとランダムなアイコンを表示するコードを書いて」
- AIの出力をMakeCodeに貼り付けて動作確認する
- コードを読んで、各行が何をしているか隣（またはビデオ通話で画面共有しながら）説明してみる
  - わからない行はAIに「この行は何をしているの？」と聞いて理解する
  - 「なんとなく動いた」ではなく「なぜ動くか話せる」状態を目指す
- うまく動かないときの修正のやり取り（AIとのデバッグ会話）

### 4. 入力センサー一覧の確認

- micro:bit の主な入力センサー一覧

| センサー | 取得できる値の例 |
| --- | --- |
| ボタン A / B | 押す／離す／長押し |
| 加速度センサー | X / Y / Z 軸の傾き、振った強さ |
| 温度センサー | 環境温度（℃） |
| 光センサー | 明るさ（0〜255） |
| マイク（v2） | 音の大きさ（dB） |

- チュートリアルで使ったセンサーと対応させて整理する

### 5. 出力の実習

- LEDマトリクスの制御
  - アイコン・文字のスクロール表示
  - 座標指定で個別LEDを点灯（`led.plot(x, y)`）
  - AIでアニメーションパターンを生成してみる
- スピーカーの制御
  - 音階の再生（`music.playTone`）
  - ビルトインメロディの再生
  - センサーの値に応じて音を変える（例：温度が高いほど音が高くなる）
- AIを使って出力プログラムを生成・修正する実践

### 6. まとめと次回予告

- Day2の振り返り（入力・出力それぞれで触ったセンサーと出力の確認）
- 次回（Day3）の予告：今日学んだ入力と出力を組み合わせてミニ作品を作る
- 事後学習の確認

## 備考

- 授業はハイブリッド形式（現地＋リモート）で実施
- MakeCode は makecode.microbit.org でブラウザから利用（インストール不要）
- micro:bit は大学で用意する（現地参加者のみ実機を使用）
- リモート参加者は MakeCode のシミュレーターで全実習を代替できる

- AIへのプロンプトは「MakeCode JavaScript で〜」と指定するとJavaScript形式で出力されやすい
