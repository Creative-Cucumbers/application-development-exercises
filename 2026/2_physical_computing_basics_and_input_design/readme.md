# Day2 フィジカルコンピューティング：基礎・入力・出力

[TOC]

## この日の位置づけ

Day1でAI時代のものづくりの全体像を掴んだ次のステップとして、実際に手を動かしてフィジカルコンピューティングの基礎を体験する。micro:bit と MakeCode を使い、「デジタルが物理世界とつながる」感覚をつかむ。

1コマ目で概念・環境構築・入力センサーを扱い、2コマ目で出力（LED・スピーカー）まで一通り体験する。1日で入力から出力まで触れることで、次回（Day3）の応用制作につながるイメージを持ちやすくする。

授業はハイブリッド形式で実施する。現地参加者は実機の micro:bit を使い、リモート参加者は MakeCode のシミュレーターで同じ実習を行う。

## 到達目標

- フィジカルコンピューティングの概念と代表的な活用事例を説明できる
- micro:bit と MakeCode の基本操作を習得し、自分でプログラムを書いて実行できる
- AI（Copilot・ChatGPT等）を使ってMakeCodeのJavaScriptコードを生成・修正できる
- AIが生成したコードが何をしているか、自分の言葉で説明できる
- micro:bit のボタン・加速度センサー・温度センサーのデータを取得できる
- LEDマトリクスとスピーカーを使った出力プログラムを組める
- AIがフィジカルコンピューティングの表現をどう拡張しているか、自分の言葉で話せる

### 1. フィジカルコンピューティングとは【講義】

デジタルと物理世界をつなぐとはどういうことか、事例と合わせて説明する。

代表的な事例：

- インタラクティブアート（teamLab, Zimoun など）
- スマート家電・IoTデバイス
- 体験型インスタレーション
- ロボット制御（センサーで障害物を検知して動きを変える、腕の動きをモーターに伝えるなど）
- 自動車・モビリティ（カメラ・LiDARで周囲を認識して走行・停止を判断する自動運転、衝突検知センサーなど）
- AIとセンサーを組み合わせた作品（音声認識 + LED、顔認識 + 物理フィードバックなど）

AI時代のフィジカルコンピューティング：

- センサーデータをAIモデルに渡すことで、より豊かなインタラクションが可能に
- コード生成AIを使えば、プログラミング経験が少なくてもプロトタイプを作れる
- 表現のアイデアを先に決め、実装はAIと協力する開発スタイル

フィジカルAI（Physical AI）の広がり：

- 近年、AIが「デジタルの世界だけ」から「物理世界に作用する」方向へ進化している
- ロボットや自律走行車、ドローンなど、センサーで環境を認識してリアルタイムに動作を決定するシステム
- NVIDIAが提唱する「Physical AI」の概念：AIが視覚・触覚・音など複数のセンサー情報を統合して判断する
- 身近な例：スマートスピーカー（音声 → AIが判断 → 照明や家電を操作）、自動運転（カメラ・LiDAR → AIが判断 → 操舵・加速・停車）
- 今日やる micro:bit の入力→処理→出力の流れは、これらの大規模なシステムと根っこは同じ

### 3. AIを使ったコード生成の実践【実習】

チュートリアルで動かしたプログラムをJavaScriptで見てみる。ブロックとJavaScriptが対応していることを確認したうえで、自分でAIにコードを生成させて動かす。

AIにコードを生成させるときのポイント：

- プロンプトに「MakeCode JavaScript で〜」と指定する
- 出力を貼り付けて動作確認する
- うまく動かないときはAIとデバッグ会話をする

あわせて、AIを「答えを出す道具」だけでなく「発想を広げる相手」として使うことも意識する。

- いきなり答えを1つ出してもらうだけでなく、複数案を出させて比べる
- 自分の興味ややってみたい体験を伝えて、案を具体化する
- 生成されたコードをそのまま使うのではなく、何をしているか説明できるようにする

**実習**：以下のプロンプトでAIにコードを生成してもらい、動かしてみる。

基本課題：

- 「MakeCode JavaScript で、ボタンAを押すたびにカウントアップしてLEDに表示するコードを書いて」
- 「MakeCode JavaScript で、micro:bit を振るとランダムなアイコンを表示するコードを書いて」
- 動いたコードを読んで、各行が何をしているか隣の人（またはビデオ通話で画面共有しながら）に説明してみる
- わからない行はAIに「この行は何をしているの？」と聞いて理解する（「なんとなく動いた」ではなく「なぜ動くか話せる」状態を目指す）

提出課題：

- 作成したプログラムの画面キャプチャを1〜2枚残す
- 実機で試した人は動作している様子を写真で1枚残す。リモート参加者はシミュレーター画面のキャプチャでよい
- 「どんな入力に対して、どんな出力が起きるか」を文章で2〜3文で説明する
  例：「ボタンAを押すと数字が1ずつ増えてLEDに表示される。micro:bit を振るとランダムなアイコンが出る。入力によって違う表示が起きるようにした」
- AIに投げたプロンプトと、修正のために追加で聞いた内容をメモとして残す

オプション課題（余裕がある人向け）：

- AIにコードを修正させてアレンジを加えてみる（例：「カウントが5になったら音を鳴らすようにして」）
- 自分でプロンプトを考えて別のプログラムを作ってみる

### 5. 出力の実習【実習】

LEDマトリクスとスピーカーでできることを説明してから実習に入る。

LEDマトリクスでできること：

- アイコン・文字のスクロール表示
- 座標指定で個別LEDを点灯（`led.plot(x, y)`）
- 明るさの変化・アニメーション

スピーカーでできること：

- 音階の再生（`music.playTone`）
- ビルトインメロディの再生
- センサーの値に応じて音を変える

**実習**：AIにコードを生成してもらい、以下をそれぞれ動かしてみる。

基本課題：

- LEDにアニメーションを表示する
- 温度センサーの値が高いほど音が高くなるプログラム
- 動いたコードを読んで、何をしているか自分の言葉で説明できるようにする

提出課題：

- LEDアニメーションまたは出力プログラムの画面キャプチャを1枚残す
- 実機で試した人は作品が動いている写真を1枚残す。リモート参加者はシミュレーター画面のキャプチャでよい
- 「どのセンサー値を使い、LEDや音がどう変化するか」を2〜3文で説明する
  例：「温度センサーの値を使って音の高さを変えた。温度が上がるほど高い音が鳴るようにした。LEDには動いていることがわかるよう簡単なアニメーションを表示した」

オプション課題（余裕がある人向け）：

- 光センサーやマイク（v2）を使ったプログラムをAIに相談しながら作ってみる
- 複数のセンサーを組み合わせてみる（例：ボタンで温度計のオン/オフを切り替える）

### 6. コードを読むための基本概念：ループ・論理・変数【講義＋実習】

入力・出力の両方を動かした今、作ったコードを改めて読み解く。AIが生成したコードを理解するために最低限知っておきたい概念を、MakeCode のブロックと対応させながら整理する。

#### 変数

- 値に名前をつけて記憶しておく仕組み
- 例：`let count = 0` → `count` という名前の箱に `0` を入れる
- センサーの値や状態を保存するときに使う（例：ボタンを何回押したか数える）
- MakeCode：「変数」カテゴリの `設定` ブロック

#### 論理（条件分岐）

- 「もし〇〇なら△△する、そうでなければ□□する」という分岐
- 例：`if (temperature > 30) { ... }` → 温度が30以上のときだけ処理を実行
- センサーの値に応じて出力を変えるときに使う
- MakeCode：「論理」カテゴリの `もし〜なら` ブロック

#### ループ（繰り返し）

- 同じ処理を何度も繰り返す仕組み
- `forever`（ずっと）ブロック：電源が入っている間ずっと繰り返す（センサーの値を常に監視する）
- `for` ループ：決まった回数だけ繰り返す（アニメーションのフレームを順番に表示するなど）
- MakeCode：「ループ」カテゴリ

**実習**：今日作ったコードをどれか1つ開いて、以下を確認してみる。

基本課題：

- コードの中でループ・論理・変数がどこで使われているか指さしてみる
- 「この `if` は何を判断しているか」「この変数は何を記憶しているか」を隣の人に説明してみる
- わからない部分はAIに「このコードの〇〇の部分は何をしているの？」と聞く

提出課題：

- 使ったコードを1つ選び、ループ・論理・変数がどこにあるかを文章で短く説明する
  例：「`count` という変数でボタンを押した回数を記録している。`if` で回数が5回になったかを判断している。`forever` で表示を繰り返している」
- AIに質問して理解した箇所があれば、その質問と要点をメモとして残す

オプション課題（余裕がある人向け）：

- AIに「変数を使ってボタンを押した回数をカウントし、5回でLEDに特別なアイコンを表示するコードを書いて」と頼んでみる
- 生成されたコードの変数・論理・ループをそれぞれ説明できるようにする
