# Day6・Day7 総合演習：作り込み・デプロイ・改善・発表

[TOC]

## この2日間の位置づけ

この総合演習では、アイデアの新しさよりも、**自分で仕組みを理解しながら、動くものを作り込むこと**を重視する。

Day6 では、画面・UI・micro:bit など、ユーザーが直接触れる部分を中心に作り、1日目のうちにデプロイまで進める。完成度よりも、まず URL で見られる状態、または動作確認できる状態にすることを優先する。

Day7 では、Day6 で作ったものをもとに改善する。必要に応じて、データベース、認証、外部 API、micro:bit 連携などを追加する。最後は全員が発表し、作品紹介だけでなく「どういう仕組みで動いているか」を説明する。

## 到達目標

- 最小限でも動くアプリまたは作品を作れる
- 作ったものをデプロイし、他の人がアクセスできる状態にできる
- UI、入力、出力、データの流れを自分の言葉で説明できる
- 授業で扱った技術を作品の中で使い、その役割を説明できる
- 他の人に見てもらい、質問やコメントをもとに改善できる
- 発表でデモを行い、作り込んだ点と仕組みを説明できる
- レポートで、自分の作品がどのように動いているかを記述できる

## 基本方針

- テーマ決定は短く済ませ、作り込みに時間を使う
- 1日目は UI とデプロイを優先する
- micro:bit は自由選択とし、使う場合は UI の一部として扱う
- 2日目は改善を中心にする
- データベース、API、認証、micro:bit は必須ではないが、できるだけ多くの技術を作品に取り入れる
- 評価では、使った技術の数だけでなく、それぞれをどのように使い、どう動いているかを理解して説明できるかを見る
- 最後は20人程度が全員発表する

## Day6：UI 実装とデプロイ

### 1コマ目：制作計画・UI設計・仕組みの整理（100分）

| 時間 | 内容                                                             |
| ---: | ---------------------------------------------------------------- |
| 10分 | 導入：2日間のゴール、作り込み重視、1日目デプロイの確認           |
| 10分 | 個人作業：作るものを1つに決める                                  |
| 15分 | 個人作業：今日作る最小機能を決める                               |
| 15分 | グループ共有：作るもの、操作方法、使う技術を説明し、質問を受ける |
| 20分 | 個人作業：画面、UI、micro:bit の使い方を決める                   |
| 15分 | 個人作業：処理の流れを書く                                       |
| 10分 | グループ共有：処理の流れを説明し、質問を受ける                   |
|  5分 | 実装開始前チェック：最初に作るものを確認する                     |

### このコマで決めること

- 何を作るか
- ユーザーは何を操作するか
- 画面または micro:bit には何が表示されるか
- 入力された値はどこで処理されるか
- どのデータを仮で持つか
- 今日デプロイする最小版は何か

### 仕様書として書き出す

作り始める前に、作るものの仕様を短い文章でまとめる。「ユーザーが何を入力し、何が返ってくるか」「画面にどう表示されるか」「どの処理がどこで動くか」を箇条書きにしたものでよい。

これを書いておくことで、実装前に仕組みを頭で整理できる。また、AI にコードを書いてもらうときの指示にそのまま使えるほか、完成後に「仕様通りに動いているか」を確認する基準にもなる。仕様が先にあることで、何を作るかがぶれにくくなる。

### 2コマ目：UI実装・修正・デプロイ（100分）

| 時間 | 内容                                                         |
| ---: | ------------------------------------------------------------ |
| 35分 | 個人作業：UI 実装、micro:bit 連携、仮データ実装              |
| 10分 | 仕組みメモ：今作った部分がどう動くかを書く                   |
| 15分 | グループ内デモ：ローカルまたは micro:bit 込みで見せる        |
| 25分 | 個人作業：デモで出た質問をもとに修正し、デプロイする         |
| 10分 | デプロイ URL 確認：グループ内でアクセスできるか確認する      |
|  5分 | 振り返り：できたこと、終わっていないこと、明日やることを書く |

### 動いたらバックアップをとる

動く状態になったタイミングで、こまめに保存しておく。壊す前に残す、が基本。

- **git を使う場合**：`git commit` でその時点を記録する。git が分からない場合は、調べるか、近くの人に聞く
- **git を使わない場合**：ファイルを `index_v2.html` のように名前をつけてコピーして残す

### Day6 の最低到達ライン

- 画面が表示される
- ボタン、入力欄、一覧、表示エリアなど、操作できる UI がある
- 仮データでもよいので、操作すると画面や状態が変わる
- デプロイ URL がある
- 他の人の端末からアクセスできる
- 自分の作品がどう動いているかを簡単に説明できる

micro:bit を使う場合は、以下のどれかを満たしていればよい。

- micro:bit を入力デバイスとして使う
- micro:bit のセンサー値を使う
- micro:bit の LED や表示を出力として使う
- Web アプリの状態と micro:bit の動きがつながっている

### Day6 の宿題

授業時間内に終わらなかったところがあれば、できる範囲で進める。

あわせて、次回の作業にすぐ入れるように、以下を整理しておく。

- できたこと
- まだ終わっていないこと
- うまく動いていないところ
- 次に直すところ
- 追加したい機能
- 発表で見せられそうなところ
- 仕組みとして説明できるところ、まだ説明があいまいなところ

### 改善の観点

- 操作方法が分かりやすいか
- 入力と出力のつながりが見えるか

### オプション技術

以下は必須ではないが、できるだけ積極的に取り入れる。すべてを使う必要はないが、授業で扱った技術を自分の作品にどう組み込めるかを考える。

- データベースへの保存
- API サーバーの実装
- 認証、ログイン
- 外部 API 連携
- micro:bit と Web アプリの連携強化

使った技術については、発表やレポートで「どこに使ったか」「何のために使ったか」「どういう仕組みで動いているか」を説明する。

## 最終成果物

最終成果物は、デプロイされた Web アプリ、ローカルで動作するアプリ、または micro:bit を含む作品とする。全員が発表でデモできる状態を目指す。

最低限必要なものは以下とする。

- 操作できる UI がある
- 入力に対して画面、データ、micro:bit などの出力が変化する
- 自分以外の人に動作を見せられる
- 使った技術と仕組みを説明できる

micro:bit、データベース、API、認証はオプションだが、できるだけ多く挑戦する。評価では、単に機能があるかだけでなく、授業で扱った技術を適切に使えているか、その仕組みを理解して説明できているかを見る。

## 評価の観点

- 動く成果物になっているか
- UI が操作でき、入力と出力の関係が分かるか
- デプロイまたは動作確認できる状態になっているか
- 授業で扱った技術を作品に取り入れているか
- micro:bit、API、データベース、認証などを使った場合、その役割を説明できるか
- コードや処理の流れを自分の言葉で説明できるか
- レビューを受けて改善した跡があるか
- 発表とレポートで、作品の仕組みを具体的に説明できているか

## レポートで書くこと

レポートでは、作品紹介だけでなく、**どういう仕組みで動いているか**を説明する。

特に、micro:bit、API、データベース、認証などを使った場合は、使ったことを書くだけでなく、作品の中でどの役割を持っているのかを説明する。

レポート作成では AI を使ってよい。むしろ、構成を整理したり、説明を分かりやすくしたりするために積極的に使ってよい。ただし、レポートに書いた内容は自分で理解している必要がある。AI が書いた文章をそのまま提出するのではなく、作品のコードや動作と照らし合わせ、自分の言葉で説明できる状態にする。

### 基本項目

- 作品名
- 作品の概要
- 使い方
- 画面、UI の構成
- 入力と出力
- データの流れ
- どの処理がどこで動いているか
- 使用した技術
- 授業で扱った技術のうち、作品に取り入れたもの
- 作り込んだ点
- うまくいかなかった点
- 次に改善したい点

### micro:bit を使った場合

- micro:bit は入力、出力、表示のどの役割を担当しているか
- Web アプリとつながっている場合、どう通信しているか
- micro:bit を使うことで何ができるようになったか

### データベースを使った場合

- 何を保存しているか
- どのタイミングで保存しているか
- どのタイミングで取得しているか
- 画面表示とデータベースの関係

### 認証を使った場合

- 何のためにログインが必要か
- ログイン状態をどう判定しているか
- ログインしていない場合に何が制限されるか
