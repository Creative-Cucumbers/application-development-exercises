# Day3 フィジカルコンピューティング：応用・組み合わせ

[TOC]

## この日の位置づけ

Day2から1週間空くため、冒頭で入力・出力の基礎を振り返るところから始める。復習を軽くこなしたうえで、Day2で習ったセンサーと出力を組み合わせた応用プログラムを体験し、後半はグループでミニ作品を制作・発表する。

「AIにコードを書いてもらいながら、自分でも何をしているか説明できる」状態を引き続き意識する。

授業はハイブリッド形式で実施。リモート参加者はMakeCodeのシミュレーターで同様に取り組む。

## 到達目標

- Day2で習った入力センサーと出力の基礎を自分の言葉で説明できる
- 入力と出力を組み合わせたインタラクションを設計・実装できる
- AIを使いながら、アイデアを短時間でプロトタイプとして動かせる
- 制作したミニ作品の動作を自分の言葉で他者に説明・デモできる

## アジェンダ案

### 0. アイスブレイク・復習

- テーマ：「Day2で一番面白かった／驚いたことは何だった？」
  - 現地・リモートともにチャットや口頭で共有（2〜3分）
  - 狙い：1週間空いた記憶をほぐしつつ場の空気を作る
- 入力・出力の簡単な振り返り（教員からクイズ形式で）
  - 例：「加速度センサーで取れるのは何の値？」「LEDの座標指定はどのメソッド？」
  - チャットで答えを書いてもらい、正解をすぐ確認する（5〜10分）

### 1. micro:bit の拡張性

- micro:bit は本体だけでなく、外部パーツを追加することで表現の幅が大きく広がる
- MakeCode の「拡張機能」からサードパーティのライブラリを追加できる
  - サーボモーター、有機ELディスプレイ、NeoPixel（フルカラーLED）、距離センサーなど
- 他のプラットフォーム（Arduino、Raspberry Pi など）と同じように拡張できる考え方
- AIとの組み合わせでさらに広がる
  - 「〇〇を制御したい」とAIに相談すると、対応する拡張ライブラリやコードを提案してもらえる
- 今日の授業では本体機能を使うが、「もっとやりたい」と思ったときの入口として知っておく

### 3. 応用例の紹介

- Day2の入力＋出力を組み合わせると何ができるか、デモで見せる
  - 傾けると表情が変わるキャラクター（加速度→LED）
  - 温度で音のテンポが変わる（温度→スピーカー）
  - ボタンで音程が変わる簡易楽器（ボタン→スピーカー＋LED）
- 組み合わせの設計の考え方
  - 「何を感知したら（入力）」→「何を表現するか（出力）」で考える
  - AIに「〇〇を感知したら△△するMakeCodeのコード」と相談する手順

### 4. 応用実習：入力＋出力を組み合わせる

- 個人で以下のプログラムを作る（AIで生成し、動作確認・説明できるまで理解する）
  - 加速度センサーの傾きに応じてLEDの表示が変わる
  - ボタンを押すたびに音が変わる
- うまくいったら自分でアレンジを加えてみる

### 5. ミニ作品制作

#### 課題条件（必須）

- 制作：個人
- 使用する入力：ボタンA/B・加速度センサー・温度センサーから **2種類以上**
- 使用する出力：LEDマトリクス・スピーカーから **両方**
- テーマ：自由（感情表現・環境モニター・ゲーム・楽器など）

#### 制作フロー

1. 個人でアイデアスケッチ（何をセンサーが検知したら、何を表現するか）を作る
2. AIに設計を相談し、コードの骨格を生成してもらう
3. MakeCode でプログラムを組んで動作確認
4. コードの各部分が何をしているか確認する（わからない箇所はAIに聞いて理解する）
5. 気に入らない部分を修正・改善
6. グループ（4〜5人）内で各自の作品をデモし合う
7. グループで全体発表する1人を選ぶ

#### アイデア例

- 傾けると表情が変わるキャラクター（加速度→LED）
- 温度で音楽のテンポが変わる環境モニター（温度→スピーカー）
- ボタンで演奏できる楽器（ボタン→スピーカー＋LED）
- 振るとランダムなアイコンと音が出るおみくじ（加速度→LED＋スピーカー）

### 6. 発表・講評

- グループ内で各自がデモし合い、代表1人を選ぶ（グループ内シェア）
- 全体発表：各グループの代表者が発表（5〜6グループ × 約4分 = 約25分）
- 発表内容：「何を表現しようとしたか」「AIをどう使ったか」を一言添える
- 全体講評：アイデアの面白さ・AIとの協働のしかた

### 7. まとめ

- Day2〜Day3の振り返り（習ったセンサーと出力の整理）
- 次回（Day4）の予告：アプリ開発へ。コーディングエージェントと開発環境
- フィジカルとデジタルをつなぐ発想は、アプリ開発でも活きることを伝える

## 事前学習

出力デバイスの例を調べ、表現の選択肢を整理する。（2時間程度）

Day2で作ったプログラムを見返し、入力・出力の動作を確認しておく。（1時間程度）

## 事後学習

入力と出力を組み合わせた際の課題点と改善案を整理する。（2時間程度）

## 成果物の例

- 応用実習のプログラム（入力＋出力の組み合わせ）
- ミニ作品のプログラム（指定センサーと出力を組み合わせたもの）
- アイデアスケッチメモ（「何を感知して、何を出力するか」の設計メモ）

## 備考

- 授業はハイブリッド形式（現地＋リモート）で実施
