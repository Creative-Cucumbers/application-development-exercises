## この日の位置づけ

冒頭で入力・出力の基礎をしっかり振り返るところから始める。単に思い出すだけでなく、「センサーで何を受け取り、どう処理し、どんな出力に変えているか」を意識しながら前回のプログラムを再現する。次に、前回紹介したブラウザ↔micro:bit 連携のコードをAIと一緒に読み解く。後半は個人でミニ作品を制作し、グループ内シェアと全体発表を行う。

「AIにコードを書いてもらいながら、自分でも何をしているか説明できる」状態を引き続き意識する。

授業はハイブリッド形式で実施。リモート参加者はMakeCodeのシミュレーターで同様に取り組む。

## 到達目標

- Day2で習った入力センサーと出力の基礎を自分の言葉で説明できる
- 入力と出力を組み合わせたインタラクションを設計・実装できる
- AIを使いながら、アイデアを短時間でプロトタイプとして動かせる
- 制作したミニ作品の動作を自分の言葉で他者に説明・デモできる

### 4. AIとのやり取り：アイデアを深掘りする【講義】

AIにいきなり「コードを書いて」と頼むのではなく、まずアイデアを深掘りしてから実装に進む流れを意識する。

AIは「答えを出す道具」だけでなく、「問いかける相棒」として使える。

- 「自分が作りたいものをはっきりさせたいので、一つずつ質問してください」と頼む
- 最初の思いつきで止まらず、「10通りの案を出して」と頼んで比べる
- 自分の経験や興味を伝えて、作品の方向性を自分らしくする

AIへの指示の仕方には大きく2種類ある。

「実装を頼む」プロンプト（コードを生成させる）：

- 「MakeCode JavaScript で、〇〇するコードを書いて」
- Day2ではこちらを中心に使った

「アイデアを深掘りする」プロンプト（考えを広げる）：

- 「micro:bit でできる面白いインタラクションのアイデアを10個出して」
- 「加速度センサーと音を組み合わせてアート作品を作るとしたら、どんな表現が考えられる？」
- 「このアイデア（〇〇）をもっと面白くするにはどうしたらいい？」
- 「〇〇という表現を実現するには、どのセンサーと出力の組み合わせが使えそう？」

おすすめの流れ：

1. まず観察ややりたい体験を言葉にする
2. アイデアを深掘りするプロンプトで選択肢を広げる
3. 気に入ったアイデアに対して実装を頼むプロンプトでコードを生成させる

プロンプト例：

- 「人が机の上で触って楽しい micro:bit 作品を考えたい。入力は加速度センサーとボタン、出力は LED と音を使う前提で、方向性を5案出して」
- 「このアイデアをもっと面白くするにはどうしたらいい？」
- 「この案の実装に必要な入力と出力を整理して。まだコードは書かず、仕組みだけ説明して」
- 「では MakeCode JavaScript で最初の試作コードを書いて」

今日のミニ作品制作では、この「深掘りしてから実装する」使い方を試してみる。

### 8. センサー・出力デバイスの注意点【講義】

実際にデバイスを使うときに知っておくと役立つことを伝える。（10分）

**電力・インターフェース**

- 拡張デバイスとつなぐには電圧・電流・通信方式（I2C、SPI、UART など）の互換性に注意
- 出力デバイスは消費電力が大きいものがある
  - Bluetooth：比較的省電力
  - Wi-Fi：電気を食う
  - モーター：さらに大きな電流が必要なことが多い
- 電源が持つかどうかを事前に確認する

**通信の安定性**

- 無線通信は必ずしも安定しない
- 「送れたかどうか」をユーザーが確認できる UI/UX の工夫が必要（LED の点滅、音、ディスプレイ表示など）

**インターフェース・ライブラリ**

- デバイスを選ぶときはインターフェース（I2C・SPI・UART など）とライブラリの有無が重要
- ライブラリや関数が用意されているデバイスは比較的簡単に使える
- ニッチなデバイスは自分でプロトコルを実装する必要があることも
- 購入前に「対応ライブラリがあるか」を確認する

**コスト・安全・安定性**

- ハードウェアは購入すると費用がかかる。試す前にAIと会話して「これで動きそうか」のあたりをつけてから買う
- 感電リスクや発熱がないか確認する（特に電流が大きいデバイス）
- 安定して動作するか、長期間使えるかも設計のうち

**作った話**

- ボタンを押すと自分と妻のスマートフォンに通知が飛ぶデバイスを作った
- 電池が3日ほどで切れてしまう問題がある
- 配線の引き回し・固定方法など「動かして終わり」ではない課題が残る
- 無線が安定しないケースもあり、「送れたかどうか」をわかりやすく伝える工夫が必要だと感じている
