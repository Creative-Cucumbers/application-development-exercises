# Day3 フィジカルコンピューティング：応用・組み合わせ

[TOC]

## この日の位置づけ

Day2から1週間空くため、冒頭で入力・出力の基礎をしっかり振り返るところから始める。単に思い出すだけでなく、「センサーで何を受け取り、どう処理し、どんな出力に変えているか」をコードとブロックの両方で再確認したうえで、Day2で習ったセンサーと出力を組み合わせた応用プログラムを体験する。後半は個人でミニ作品を制作し、グループ内シェアと全体発表を行う。

「AIにコードを書いてもらいながら、自分でも何をしているか説明できる」状態を引き続き意識する。

授業はハイブリッド形式で実施。リモート参加者はMakeCodeのシミュレーターで同様に取り組む。

## 到達目標

- Day2で習った入力センサーと出力の基礎を自分の言葉で説明できる
- 入力と出力を組み合わせたインタラクションを設計・実装できる
- AIを使いながら、アイデアを短時間でプロトタイプとして動かせる
- 制作したミニ作品の動作を自分の言葉で他者に説明・デモできる

## アジェンダ案

### 0. アイスブレイク・復習【全体参加】

Day2からの記憶をほぐしつつ場の空気を作る。

テーマ：「Day2で一番面白かった／驚いたことは何だった？」

- 現地・リモートともにチャットや口頭で共有（2〜3分）

### 1. 復習：入力・出力・ループ・論理・変数【講義＋実習】

1週間空いたため、Day2で扱った内容を手を動かしながら思い出す。コードとブロックを対応させて確認し、「入力」「処理」「出力」を言葉で説明できる状態まで戻す。

#### 復習実習

Day2で作ったコードをどれか1つ開いて、以下を確認する。

基本課題：

- まず「このプログラムは何を入力として受け取り、何を出力しているか」を1文で書く
- ループ・論理・変数がコードのどこにあるか指さす
- 「入力はどこか」「出力はどこか」をコード上で確認する
- 1つ変更を加えてみる（例：しきい値の数字を変える、表示するアイコンを変えるなど）

提出課題：

- 見直したコードの画面キャプチャを1枚残す
- 「何を入力として受け取り、何を出力しているか」を1〜2文で書く
  例：「ボタンAの入力を受け取ると、LEDの表示が変わるプログラムを見直した。入力に応じて出力が切り替わる仕組みを確認した」
- 変更した箇所と、変えた結果どうなったかを1〜2文で書く
  例：「しきい値の数字を変えたら、反応するタイミングが早くなった。表示するアイコンも変えて、変化がわかりやすくなった」

オプション課題（余裕がある人向け）：

- Day2のオプション課題まで到達しなかった人は、ここで試してみる
- 2つの入力を組み合わせた簡単なプログラムにしてみる

### 2. AIとのやり取り：アイデアを深掘りするプロンプト【講義】

Day3では、AIにいきなり「コードを書いて」と頼むのではなく、まずアイデアを深掘りしてから実装に進む流れを意識する。

AIへの指示の仕方には大きく2種類ある。

「実装を頼む」プロンプト（コードを生成させる）：

- 「MakeCode JavaScript で、〇〇するコードを書いて」
- Day2ではこちらを中心に使った

「アイデアを深掘りする」プロンプト（考えを広げる）：

- 「micro:bit でできる面白いインタラクションのアイデアを10個出して」
- 「加速度センサーと音を組み合わせてアート作品を作るとしたら、どんな表現が考えられる？」
- 「このアイデア（〇〇）をもっと面白くするにはどうしたらいい？」
- 「〇〇という表現を実現するには、どのセンサーと出力の組み合わせが使えそう？」

おすすめの流れ：

1. まず観察ややりたい体験を言葉にする
2. アイデアを深掘りするプロンプトで選択肢を広げる
3. 気に入ったアイデアに対して実装を頼むプロンプトでコードを生成させる

プロンプト例：

- 「人が机の上で触って楽しい micro:bit 作品を考えたい。入力は加速度センサーとボタン、出力は LED と音を使う前提で、方向性を5案出して」
- 「このアイデアをもっと面白くするにはどうしたらいい？」
- 「この案の実装に必要な入力と出力を整理して。まだコードは書かず、仕組みだけ説明して」
- 「では MakeCode JavaScript で最初の試作コードを書いて」

今日のミニ作品制作では、この「深掘りしてから実装する」使い方を試してみる。

### 3. micro:bit の拡張性【講義】

micro:bit は本体だけでなく、外部パーツを追加することで表現の幅が大きく広がる。

- MakeCode の「拡張機能」からサードパーティのライブラリを追加できる
  - サーボモーター、有機ELディスプレイ、NeoPixel（フルカラーLED）、距離センサーなど
- 他のプラットフォーム（Arduino、Raspberry Pi など）と同じように拡張できる考え方
- AIとの組み合わせでさらに広がる（「〇〇を制御したい」とAIに相談すると、対応する拡張ライブラリやコードを提案してもらえる）
- 今日の授業では本体機能を使うが、「もっとやりたい」と思ったときの入口として知っておく

---

### 4. 応用例の紹介【講義・デモ】

Day2の入力＋出力を組み合わせると何ができるか、デモで見せる。

デモ例：

- 傾けると表情が変わるキャラクター（加速度→LED）
- 温度で音のテンポが変わる（温度→スピーカー）
- ボタンで音程が変わる簡易楽器（ボタン→スピーカー＋LED）

組み合わせの設計の考え方：

- 「何を感知したら（入力）」→「何を表現するか（出力）」で考える
- AIにはまず「どんな反応がありえるか」「どの入力と出力が合いそうか」を相談し、そのあとで「〇〇を感知したら△△するMakeCodeのコード」と実装を頼む

### 5. 応用実習：入力＋出力を組み合わせる【実習】

**実習**：個人で以下のプログラムを作る。AIで生成し、動作確認・説明できるまで理解する。

基本課題：

- 加速度センサーの傾きに応じてLEDの表示が変わるプログラム
- ボタンを押すたびに音が変わるプログラム

提出課題：

- 作成したプログラムの画面キャプチャを1〜2枚残す
- 実機で試した人は動作の様子を写真で1枚残す。リモート参加者はシミュレーター画面のキャプチャでよい
- 「どの入力に対して、どの出力がどう変わるか」を2〜3文で説明する
  例：「micro:bit を右に傾けるとLEDの顔が笑顔に変わる。左に傾けると別の顔になる。ボタンを押すと音が変わり、入力ごとに出力が変化するようにした」
- AIに相談したプロンプトをメモとして残す

オプション課題（余裕がある人向け）：

- 上記2つを組み合わせたプログラムを作る
- 光センサーやマイクを追加して3種類以上のセンサーを使ってみる

### 6. ミニ作品制作【実習】

Day2〜Day3で学んだ入力と出力を使って、テーマ自由のミニ作品を個人で制作する。

課題条件（必須）：

- 制作：個人
- 使用する入力：ボタンA/B・加速度センサー・温度センサーから2種類以上
- 使用する出力：LEDマトリクス・スピーカーの両方
- テーマ：自由（感情表現・環境モニター・ゲーム・楽器など）

アイデア例：

- 傾けると表情が変わるキャラクター（加速度→LED）
- 温度で音楽のテンポが変わる環境モニター（温度→スピーカー）
- ボタンで演奏できる楽器（ボタン→スピーカー＋LED）
- 振るとランダムなアイコンと音が出るおみくじ（加速度→LED＋スピーカー）

**実習**：以下の流れで制作する。

制作フロー：

1. 作りたい体験や雰囲気を短く言葉にする
2. AIにアイデアを深掘りするプロンプトを投げて、表現のアイデアを広げる（例：「micro:bit の加速度センサーと音を組み合わせて面白い表現を5つ提案して」）
3. 気に入ったアイデアを1つ選び、個人でアイデアスケッチを作る
4. AIに実装を依頼するプロンプトでコードの骨格を生成してもらう
5. MakeCode でプログラムを組んで動作確認する
6. コードの各部分が何をしているか確認する（わからない箇所はAIに聞いて理解する）
7. 気に入らない部分を修正・改善する
8. グループ（4〜5人）内で各自の作品をデモし合う
9. グループで全体発表する1人を選ぶ

提出課題：

- ミニ作品の MakeCode 画面キャプチャを1〜2枚残す
- 実機で試した人は完成作品の写真を1枚以上残す。リモート参加者はシミュレーター画面のキャプチャでよい
- 「何を作ろうとしたか」「どの入力と出力を使ったか」「どこをAIに助けてもらったか」を3〜5文で書く
  例：「傾きとボタンで遊べる小さな楽器を作ろうとした。入力は加速度センサーとボタンA/B、出力はLEDとスピーカーを使った。AIにはアイデアを広げる相談と、最初のコード作成を手伝ってもらった。自分では音の変化と表示の調整を行った」
- 使ったプロンプトや、AIとのやり取りで案を深掘りした記録をメモとして残す

オプション課題（余裕がある人向け）：

- 作品に追加機能を加えてみる（例：ボタンでモードを切り替える、LEDと音を連動させるなど）
- 2台の micro:bit を無線（Radio）でつないで、片方の入力がもう片方の出力に反映されるものを作る

---

### 7. 発表・講評【全体参加】

- グループ内で各自がデモし合い、代表1人を選ぶ（グループ内シェア）
- 全体発表：各グループの代表者が発表（5〜6グループ × 約4分 = 約25分）
- 発表内容：「何を表現しようとしたか」「AIをどう使ったか」を一言添える
- 全体講評：アイデアの面白さ・AIとの協働のしかた

---

### 8. まとめ【講義】

- Day2〜Day3の振り返り（習ったセンサーと出力の整理）
- Day2のシリアル通信デモの振り返り：micro:bit がブラウザのアプリとつながる仕組みを改めて確認する
- 次回（Day4）の予告：アプリ開発へ。コーディングエージェントと開発環境
- フィジカルとデジタルをつなぐ発想が、アプリ開発でも活きてくることを伝える

## 事前学習

出力デバイスの例を調べ、表現の選択肢を整理する。（2時間程度）

Day2で作ったプログラムを見返し、入力・出力の動作を確認しておく。（1時間程度）

## 事後学習

入力と出力を組み合わせた際の課題点と改善案を整理する。（2時間程度）

## 成果物の例

- 応用実習のプログラム（入力＋出力の組み合わせ）
- ミニ作品のプログラム（指定センサーと出力を組み合わせたもの）
- プログラムや動作のスクリーンショット、実機写真
- アイデアスケッチメモ（「何を感知して、何を出力するか」の設計メモ）
- 作品説明メモ（入力・出力・意図・改善点の説明）
- AIとのやり取りのメモ（使ったプロンプトと修正の記録）

## 備考

- 授業はハイブリッド形式（現地＋リモート）で実施
- micro:bit は大学で用意する（現地参加者のみ実機を使用）
- MakeCode は makecode.microbit.org でブラウザから利用（インストール不要）
- リモート参加者は MakeCode のシミュレーターで全実習を代替できる
- ミニ作品は個人制作。グループは発表代表者を選ぶためのもの（4〜5人）
