# Day4 講師ガイド

このファイルは readme.md（学生向け配布資料）の補足として、授業を進めるための意図・タイムライン・運営上の注意点をまとめたものです。学生には配布しない。

---

## 授業の設計意図

### 1コマ目の狙い

コーディングエージェントを「概念」として整理したうえで、実際に Gemini CLI を使って投票アプリを作る体験をさせる。最後に「データが消える」体験を意図的に作り出し、2コマ目への問いを持たせる。

### 2コマ目の狙い

「消えた」という具体的な体験を起点に、データ・DB・バックエンドの必要性を自然に導く。最終的に「投票アプリのデータが消えなくなった」という成功体験を授業内で完結させ、Day5（バックエンド本格化）への期待感を作る。

## 各セクションの進め方・狙い

### 0. アイスブレイク

- **狙い**: 「アプリを使う側」から「アプリを作る側」へ視点を切り替えるきっかけにする
- **進め方**: Slack の投稿をいくつかピックアップして短くコメントする

---

### 1. 最近のニュース

- ⚠️ **授業直前に内容を更新すること**。readme.md はプレースホルダーのまま。
- AIコーディングツールに関する最新動向を5分で紹介する

---

### 3. セットアップ確認

- **事前準備**: 事前学習として各ソフトウェアのインストールを済ませてくるよう、前回授業または Slack で案内しておく
- **当日の運営**: 全員の動作確認を行い、詰まっている学生はその場でフォローする。全員が揃うまで次に進まない

---

### 5. Git in VSCode

- **実習の進め方**: プロジェクトフォルダを VSCode で開き、git init → initial commit まで全員で一緒に実施する。画面を見せながら進める

---

### 7. 「消えた」問題の確認

- **1コマ目→2コマ目の橋渡し**: 「このデータを残すには、どこかに保存する必要がある」という問題提起で締め、2コマ目への問いを持たせる
- **ポイント**: 学生が自分で問いに気づけるよう、教員が結論を言いすぎない

---

### 8. なぜデータが重要か

- **時間配分**: 答え合わせ 2分 + 問いかけ 3分 + まとめ・ゴール提示 10分 = 計15分
- **答え合わせ（2分）**: Slack の回答をいくつかピックアップしてコメントし、「JavaScript の変数はメモリ上にある、ページを閉じると消える」でまとめる
- **問いかけ（3分）**: 回答をいくつか紹介しながら「どのアプリも、データをどこかに書き込んで残している」（＝永続化）につなぐ
- **まとめ（10分）**: 今日 2 コマ目のゴールを明示して学生に方向性を共有する

---

### 9. データの種類

- **時間配分**: 構造化データ実習 12分 + 非構造化データ・AI 概要 3分 = 計15分
- **非構造化データの扱い**: Day6 以降で詳しく扱うため、今日は「AI で変換できる」という事実だけ押さえる。深入りしない

---

### 10. SQLite とは・なぜサーバーが必要か

- **時間配分**: JSON 限界 3分 + SQLite 特徴 3分 + サーバー必要性 4分 = 計10分
- **server.js の概要説明**: コードを読ませる必要はない。役割の概要（require / app.post / app.get / res.json）だけ伝え、セクション 11B の前置きとする

---

### 11A. SQLite 実践（sqliteonline.com）

- **時間配分**: フェーズ1（全員同期）15分 + フェーズ2（Gemini CLI）5分 = 計20分
- **フェーズ1の運営**: 全員が CREATE TABLE → INSERT → SELECT GROUP BY まで完了してから次に進む。詰まっている学生はその場でフォロー

---

### 11B. Gemini CLI でミニサーバーを作る

- **時間配分**: パッケージ導入 5分 + サーバー生成 5分 + index.html 修正 10分 + 動作確認 10分 = 計30分
- **よくあるトラブルと対処**:
  - `better-sqlite3` のビルドエラー（Mac/Windows で起きやすい）→ 「better-sqlite3 の代わりに sqlite3 パッケージを使って同じコードを書き直して」というプロンプトを手元に用意しておく
  - ポートが使用中 → `3000` を `3001` に変えて再起動するよう案内
  - fetch がエラーになる → server.js に `express.static('.')` があるか確認するよう促す
- **バッファ**: 15分以上遅延した場合は「サーバー起動まで全員で確認、fetch 修正は事後学習」としてセクション 12 を 5 分に短縮

---

### 12. まとめ・Day5 予告

- **自己チェックの活用**: 「怪しい」「まだわからない」が多かった項目は Day5 の冒頭でフォローする。チェック結果を見てから次回冒頭の補足内容を決める
- **詰まりポイントの活用**: Slack に書いてもらった内容を Day5 の冒頭資料に反映する
