### 科目ルーブリック［評価基準］

1 知識・情報収集力（30%）
S 開発環境、データベース、フィジカルコンピューティング、Webアプリ開発に関する基礎知識を理解した上で、AIの出力も検証しながら必要な情報を収集・比較し、適切な技術や手法を選択して学習や制作に活用することができる。
A 開発環境、データベース、フィジカルコンピューティング、Webアプリ開発に関する基礎知識を理解し、AIも活用しながら必要な情報を収集して学習や制作に活用することができる。
B 開発環境、データベース、フィジカルコンピューティング、Webアプリ開発に関する基本事項を理解し、資料やAIの補助を参照しながら必要な情報を集めて課題に取り組むことができる。
C 開発環境、データベース、フィジカルコンピューティング、Webアプリ開発に関する基本事項を一部理解し、支援を受けながら必要な情報を探して課題に取り組むことができる。
D 開発環境、データベース、フィジカルコンピューティング、Webアプリ開発に関する基本事項の理解が不十分であり、必要な情報の収集や活用に継続した支援を要する。

2 論理的思考力（40%）
S フロントエンド、バックエンド、データベース、入出力処理の関係を理解した上で、AIの提案を適切に取捨選択しながら課題の原因分析や改善まで含めて、動作の安定したアプリケーションを論理的に設計・実装することができる。
A フロントエンド、バックエンド、データベース、入出力処理の関係を整理し、AIを補助的に活用しながら動作するアプリケーションを論理的に設計・実装することができる。
B フロントエンド、バックエンド、データベース、入出力処理の基本的な関係を理解し、資料やAIの補助を参照しながら基本的なアプリケーションを実装することができる。
C フロントエンド、バックエンド、データベース、入出力処理の関係を一部理解し、支援を受けながらアプリケーションの一部を設計・実装することができる。
D フロントエンド、バックエンド、データベース、入出力処理の関係の理解が不十分であり、アプリケーションの設計・実装に継続した支援を要する。

3 発想・構想力（30%）
S 課題や目的、利用場面に応じた着想をもとに、AIも活用して発想を広げ、設計・試作・検証・改善を通して独自性と実現性を両立した提案や制作物に発展させることができる。
A 課題や目的に応じてAIも活用しながらアイデアを具体化し、設計・試作・検証・改善を通して実現可能な提案や制作物に発展させることができる。
B 与えられたテーマに沿ってアイデアをまとめ、資料やAIの補助を参照しながら基本的な試作と改善を行うことができる。
C 与えられたテーマに沿ってアイデアを一部整理し、支援を受けながら試作や改善の一部を行うことができる。
D アイデアの具体化や構想の発展が不十分であり、試作や改善に継続した支援を要する。

### 授業計画

オリエンテーション
【事前学習】シラバスおよび授業の概要資料を読み、授業全体の目的と進め方を確認する (1時間程度)
【事後学習】オリエンテーションで説明された評価方法や受講の心構えを踏まえ、自身の目標を設定して学習計画を立てる (1時間程度)

AIとものづくりに与える影響
【事前学習】生成AIやノーコードツールが制作に与える影響について調べる（1時間程度）
【事後学習】ワークショップで出た気づきを整理し、自分の学習目標を言語化する（1時間程度）

フィジカルコンピューティング：基礎
【事前学習】フィジカルコンピューティングの事例を2つ調べ、概要をまとめる（2時間程度）
【事後学習】マイコンのセットアップ手順を整理し、再現できるようにする（2時間程度）

フィジカルコンピューティング：入力設計
【事前学習】センサーの種類と取得できる情報を調べる（2時間程度）
【事後学習】取得データの扱い方と注意点を整理する（2時間程度）

フィジカルコンピューティング：出力設計
【事前学習】出力デバイスの例を調べ、表現の選択肢を整理する（2時間程度）
【事後学習】出力の制御方法と安全面の配慮をまとめる（2時間程度）

フィジカルコンピューティング：組み合わせる
【事前学習】入力と出力の組み合わせパターンを考える（2時間程度）
【事後学習】組み合わせた際の課題点と改善案を整理する（2時間程度）

アプリ開発：コーディングエージェントと開発環境
【事前学習】使用する開発環境の基本操作を確認する（2時間程度）
【事後学習】コーディングエージェントの活用手順と留意点をまとめる（2時間程度）

アプリ開発：フロントエンド設計
【事前学習】UIの基礎を復習する（2時間程度）
【事後学習】画面設計とデータ構造の関係を整理する（2時間程度）

アプリ開発：バックエンドとデータベース
【事前学習】バックエンドとデータベースの役割を調べる（2時間程度）
【事後学習】データ設計の要点とAPI連携の流れを整理する（2時間程度）

アプリ開発：認証とデバッグ
【事前学習】認証・権限の基本用語を調べる（3時間程度）
【事後学習】デバッグ手順とログの見方を整理する（3時間程度）

総合演習：テーマ設計
【事前学習】制作テーマ案を作り、必要な機能を洗い出す（3時間程度）
【事後学習】フィードバック内容を整理し、改善計画を作成する（3時間程度）

総合演習：実装
【事前学習】実装計画とスケジュールを見直す（3時間程度）
【事後学習】実装の進捗と課題点を整理する（3時間程度）

総合演習：レビューと改善
【事前学習】レビューの観点を整理し、評価基準を確認する（3時間程度）
【事後学習】レビュー結果を反映し、最終調整点をまとめる（3時間程度）

最終プレゼンテーションと講評
【事前学習】デモと発表資料を整え、リハーサルを行う（3時間程度）
【事後学習】講評を踏まえた振り返りレポートを作成する（3時間程度）
